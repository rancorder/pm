# Arent Priority Domains

PM Prototype OSで、Arentとの親和性を踏まえて優先的に深掘りする建設業ドメイン。

## 優先順位

```text
S: BIM / Revit / IFC / 設計レビュー
A: 図面・仕様書・議事録RAG
A: 配筋・構造設計ルール支援
A: プラント設計支援
B: 点群→BIM/IFC
B: 工程・日報・写真連携
B: 見積・積算支援
C: 安全AI / 現場CV
C: 維持管理/FM
```

## S: BIM / Revit / IFC / 設計レビュー

### 理由

Arentの既存実績・プロダクト領域と最も近い。

### PMが狙う課題

- Revit作業の属人化
- BIM属性不足
- IFC納品チェック
- 図面・モデル・仕様の不整合
- 設計レビューの標準化

### MVP例

- IFC属性欠損チェック
- Revit操作AI Agentモック
- BIMモデルレビューリスト生成
- IDSドラフト生成支援

### ヒアリングで聞くこと

- Revitで一番面倒な操作は何か？
- BIM属性は誰が、いつ、何のために入力しているか？
- モデルチェックの基準は明文化されているか？
- 納品前に必ず確認する項目は何か？

## A: 図面・仕様書・議事録RAG

### 理由

顧客の業種を問わず導入しやすく、PMの要件定義支援にも直結する。

### PMが狙う課題

- 資料探索コスト
- 版管理の混乱
- 議事録の決定事項未反映
- 図面と仕様書のズレ

### MVP例

- 顧客資料RAG
- 議事録→変更管理表
- 図面・仕様書の不整合候補
- 次回確認事項自動抽出

## A: 配筋・構造設計ルール支援

### 理由

設計自動化・専門知識の形式知化と相性が良い。

### PMが狙う課題

- 設計ルール確認が重い
- 過去判断が再利用されない
- 若手と熟練者の差が大きい
- チェック観点が属人化

### MVP例

- 設計ルールRAG
- 配筋チェック観点リスト
- 過去案件類似検索
- NG候補提示

## A: プラント設計支援

### 理由

ArentのPlantStream系との親和性が高い。

### PMが狙う課題

- 設計ルール確認
- P&ID、3D、仕様書の関係整理
- 配管・機器・干渉・変更管理
- 設計条件の引き継ぎ

### MVP例

- プラント設計ルールRAG
- 変更指示→影響範囲抽出
- 仕様書・機器リスト検索
- 配管ルート検討支援モック

## B: 点群→BIM/IFC

### 理由

改修・維持管理・既存施設DXで価値が出るが、技術検証負荷が高い。

### MVP例

- 点群から壁・床候補抽出
- 既存図面との差分表示
- IFC化可能性レポート

## B: 工程・日報・写真連携

### 理由

施工現場で課題は大きいが、Arentの中核BIM/設計領域からは少し離れる。

### MVP例

- 工程表×日報差分
- 週次現場サマリ
- 写真分類

## B: 見積・積算支援

### 理由

BIM数量・図面・BOQとつながれば強いが、単価や会社別ルールの差分が大きい。

### MVP例

- 見積比較表生成
- BOQ差分
- 数量拾い補助

## C: 安全AI / 現場CV

### 理由

価値は大きいが、現場環境差・誤検知・責任範囲が重い。

### MVP例

- KYシート生成
- ヒヤリハット分類
- 写真から危険候補提示

## C: 維持管理/FM

### 理由

BIMの出口として有望。ただし、初期PM業務では設計・プロトタイプ化より遠い場合がある。

### MVP例

- 設備台帳RAG
- 点検履歴検索
- BIM/設備ビューア

## べーさん向け最初の勝ち筋

最初に狙うべきはこれ。

```text
顧客ヒアリング
↓
図面・仕様書・議事録の情報整理
↓
BIM/Revit/IFCに絡む不整合や属人作業を抽出
↓
1画面のMVPに落とす
↓
Claude Code / Codexでプロトタイプ化
```

## 初回提案テーマ候補

1. BIM/IFCモデルチェックAI
2. 図面・仕様書・議事録RAG
3. Revit操作AI Agentモック
4. 議事録→設計変更ログ自動化
5. 配筋/設計ルール検索AI
