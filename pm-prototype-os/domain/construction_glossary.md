# Construction Glossary

PM Prototype OSで使う建設業ドメイン用語集。

## プロジェクト・契約

| 用語 | 意味 | PMでの見方 |
|---|---|---|
| 発注者 | 工事や設計を依頼する主体 | 最終意思決定者。要件・予算・納期を持つ |
| 元請 | 発注者から直接請け負う会社 | 全体責任・工程・品質・安全を管理 |
| 下請 | 元請から一部工事を請け負う会社 | 実作業・専門工種の情報源 |
| 協力会社 | 下請・専門業者 | 現場実態を持つ重要ステークホルダー |
| 請負契約 | 工事完成を約束する契約 | 範囲・責任・変更条件が重要 |
| 追加工事 | 契約後に追加される作業 | 仕様変更・見積・承認フローが重要 |
| 変更契約 | 契約内容変更の合意 | AIで差分・根拠整理の余地あり |

## 設計・BIM

| 用語 | 意味 | PMでの見方 |
|---|---|---|
| 基本設計 | 大枠の設計 | 顧客要求を形にする初期フェーズ |
| 実施設計 | 施工できる詳細設計 | 図面・仕様の整合が重要 |
| 施工図 | 現場で施工するための詳細図 | 元設計との差分管理が重要 |
| BIM | 建物情報モデル | 形状だけでなく属性・数量・関係性が価値 |
| CIM | 土木分野の3Dモデル活用 | 土木・インフラ系で重要 |
| Revit | AutodeskのBIMソフト | Arent領域との親和性が高い |
| IFC | BIMデータ交換標準 | OSS・AI解析の入口になりやすい |
| 属性情報 | 部材名、材料、寸法、性能など | チェック・積算・維持管理の鍵 |
| IDS | BIM情報要求を機械可読化する仕様 | AI生成・検証の新テーマ |

> ISO 19650 / CDE / BCF / COBie / Digital Twin / buildingSMART / Civil 3D / Navisworksなど、標準規格・周辺ツールは `construction_standards.md` に分離して整理している。

## 積算・見積

| 用語 | 意味 | PMでの見方 |
|---|---|---|
| 積算 | 数量と単価から工事費を算出 | 拾い漏れ・単価差・根拠が課題 |
| 数量拾い | 図面から数量を抽出 | BIM/AI活用余地が大きい |
| BOQ | Bill of Quantities、数量明細 | 見積比較の軸 |
| 単価 | 材料・労務・機械などの価格 | 市況・地域差・条件差に注意 |
| 歩掛 | 作業に必要な労務・機械などの標準量 | 原価・工程の根拠 |
| NET金額 | 実質原価・仕入れに近い金額 | 粗利管理で重要 |
| VE | Value Engineering | 価値を落とさずコスト最適化 |
| CD | Cost Down | 単純なコスト削減 |

## 工程・施工管理

| 用語 | 意味 | PMでの見方 |
|---|---|---|
| 工程表 | 作業スケジュール | 遅延要因と依存関係が重要 |
| クリティカルパス | 全体工期に直結する作業経路 | AIで遅延リスク検知可能 |
| 出来高 | 工事の進捗量 | 支払・進捗管理と関係 |
| 出来形 | 完成した形状・寸法など | 検査・品質記録の対象 |
| 施工計画書 | 工事の進め方をまとめた書類 | 安全・品質・工程の前提 |
| KY | 危険予知活動 | 安全管理の現場文化 |
| ヒヤリハット | 事故未満の危険事象 | 予防AIの材料 |
| 是正 | 不備を直すこと | 検査・写真・担当者管理が必要 |

## 品質・検査

| 用語 | 意味 | PMでの見方 |
|---|---|---|
| 品質管理 | 設計・仕様通りに施工する管理 | 証跡と基準が重要 |
| 検査記録 | 検査結果の記録 | AIチェック・未完了検知に使える |
| ミルシート | 鋼材等の品質証明書 | 材料証跡の代表例 |
| トレーサビリティ | 履歴追跡性 | 材料・施工・検査の紐付け |
| 竣工図 | 完成後の実態を反映した図面 | 維持管理に直結 |
| 維持管理 | 引渡後の点検・修繕 | BIM/FM連携の対象 |

## 現場データ

| 用語 | 意味 | PMでの見方 |
|---|---|---|
| 日報 | その日の作業記録 | 工程・出来高・労務の実績データ |
| 写真管理 | 施工状況を写真で残す業務 | AI分類・撮影漏れ検知の余地大 |
| 点群 | 3Dスキャンデータ | 既存建物・出来形・BIM化に使える |
| ドローン測量 | ドローンで現場を計測 | 土木・進捗・出来形で有効 |
| センサー | IoT計測機器 | 安全・環境・設備監視に使える |

## PM要件定義の注意

建設業では、同じ言葉でも立場により意味が変わる。

例：

- 「図面チェック」＝設計者は整合、施工者は施工可否、発注者は契約範囲を見る
- 「進捗」＝現場は作業完了、本社は出来高、発注者は支払条件を見る
- 「品質」＝設計仕様、材料証明、検査記録、写真証跡が混ざる

PMは必ず「誰にとっての何か」を確認する。
