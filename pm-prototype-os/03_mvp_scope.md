# 03 MVP Scope

プロトタイプを超短時間で作るためのMVP切り出しルール。

## MVPの定義

MVPは最小機能ではなく、最小検証単位。

```text
何を検証するか
↓
誰が判断するか
↓
どのデータで試すか
↓
何が出れば成功か
```

## MVP候補の3分類

### A. Demo MVP

顧客にイメージを掴ませるための見た目重視プロトタイプ。

- 画面だけ作る
- ダミーデータで動かす
- 操作体験を確認する

### B. Data MVP

実データで価値を検証するプロトタイプ。

- PDF/Excel/IFC/画像を読み込む
- 抽出・分類・検索を試す
- 精度と速度を見る

### C. Workflow MVP

業務フローに組み込めるかを検証するプロトタイプ。

- 入力者
- 承認者
- 出力先
- 通知
- 修正ループ

| B案 |  |  |  |  |  |
| C案 |  |  |  |  |  |

## 捨てるべきもの

初期MVPでは原則捨てる。

- ログイン
- 権限管理
- 完璧なDB設計
- 完全自動化
- 大量データ対応
- 高度なデザイン
- 例外処理の網羅

## 残すべきもの

- 顧客が価値を判断できる画面
- 実データに近い入力
- Before/Afterがわかる出力
- 次の議論が進む結果

## Arent領域でのMVP例

### BIMチェックMVP

IFCファイルを読み込み、部材一覧と属性欠損を表示する。

### Revit操作MVP

自然言語コマンドを受け取り、実行予定操作の手順を表示する。

### 図面RAG MVP

PDF図面・仕様書から、特定部材や条件を検索して回答する。

### 設計レビューMVP

議事録・図面・仕様書の不整合候補を一覧表示する。
