# Tech Card: Speckle（BIMデータのバージョン管理）

## 技術名

Speckle / BIMデータのオブジェクトベース版管理・相互運用プラットフォーム

## 参照元

- GitHub: https://github.com/specklesystems
- Docs: https://speckle.guide/

## 何ができるか

Revit・Rhino・Grasshopper・AutoCAD・Civil 3D・ArchiCADなど主要AECツールから、3Dモデルとその属性情報を「オブジェクト」としてリアルタイムに送受信できるプラットフォーム。ファイルではなくオブジェクト単位でバージョン管理するため、「Gitのようなもの」として説明すると伝わりやすい。ライセンスはApache 2.0。

## 解決できる顧客課題

- 図面・BIMモデルの最新版がどれか分からない
- 複数ツール（Revit⇔Rhino⇔Civil 3D等）間でのモデル受け渡しが手作業
- 変更履歴・差分が追えない

## Arent領域との親和性

- BIM: 高
- Revit: 高（公式コネクタあり）
- IFC: 中（IFCインポート可能。ただしSpeckle独自のオブジェクトモデルが主）
- 配筋: 低〜中
- プラント: 中（Civil 3D、GISコネクタあり）
- 図面/仕様書RAG: 低

## 入力

- Revit/Rhino/Civil 3D等からのモデルストリーム
- IFC/STL/OBJファイル

## 出力

- バージョン管理されたオブジェクトグラフ
- Web上の3Dビューア
- 差分（コミット間のジオメトリ・属性差分）
- GraphQL API経由の構造化データ

## MVPでの使い方

顧客に「図面の最新版が分からない」という課題があれば、Speckleのパブリックサーバー（無料枠あり）に1案件分のモデルをアップロードし、バージョン履歴と差分をWebビューアで見せるだけでも価値検証になる。自前サーバー構築は本番化フェーズまで不要。

## プロトタイプ案

```text
Revit/Rhino Connector
↓
Speckle Stream（バージョン管理）
↓
Web Viewer（3D差分表示）
↓
GraphQL API（PM Prototype OS側から差分取得）
```

## 実装難易度

中（パブリッククラウドサーバー利用なら低。自前ホスティングは中〜高）

## 注意点

- ライセンス：サーバー/フロントエンドはApache 2.0。パブリッククラウドサーバー利用は無料枠に容量制限あり
- セキュリティ：顧客の機密モデルをパブリッククラウドに上げてよいか、事前に必ず確認する
- データ品質：Revit側のモデリング品質にそのまま依存する
- 本番化リスク：自前サーバーホスティングは運用負荷が発生する

## 顧客に聞くべき質問

1. 現在Revit以外に何のツールを併用しているか（Rhino/Civil 3D/ArchiCAD等）？
2. モデルの受け渡しは今どうやっているか（メール添付/共有フォルダ/クラウド）？
3. 機密性の高いモデルをクラウドサービスに上げることに抵抗があるか？

## エンジニアに聞くべき質問

1. パブリッククラウドサーバーで初期PoCを回してよいか、自前ホスティングが必須か？
2. Revitコネクタのインストール権限が顧客環境にあるか？
3. IFC経由の連携で十分か、ネイティブコネクタが必要か？
