# Tech Card: IFC Analysis

## 技術名

IFC解析 / OpenBIMデータ抽出

## 参照候補

- IfcOpenShell
- BlenderBIM
- IFC model checker系OSS

## 何ができるか

IFCファイルから、部材・属性・階層・関係性・数量などを抽出し、BIMモデルを業務データとして扱えるようにする。

## 解決できる顧客課題

- BIMモデルの中身を人手で確認している
- 属性欠損や命名不備を見逃す
- 図面・仕様書・BIMモデルの整合確認が重い
- 設計レビューが属人化している

## Arent領域との親和性

- BIM: 高
- Revit: 高
- IFC: 最高
- 配筋: 中
- プラント: 中〜高
- 図面/仕様書RAG: 高

## 入力

- IFCファイル
- RevitからエクスポートしたBIMデータ
- チェックルール

## 出力

- 部材一覧
- 属性一覧
- 欠損候補
- 不整合候補
- 数量情報
- レビュー対象リスト

## MVPでの使い方

IFCファイルをアップロードすると、部材一覧・属性欠損・確認優先度をWeb画面に表示する。

**実装済みスターターあり**：`starters/ifc-mvp/` に、この内容をそのまま動く形にしたものがある（web-ifc-viewer使用、MIT）。実際のIFCファイルで動作検証済み（壁・ドア・窓等の検出、属性欠損候補の自動表示まで確認済み）。「顧客に見せる1枚」が必要なときはまずここを`npm install && npm run dev`すれば数分で立ち上がる。

## プロトタイプ案

```text
IFC Upload
↓
Model Summary
↓
Element Table
↓
Missing Attribute Check
↓
Review Report
```

## 実装難易度

中

## 注意点

- IFCファイルの品質に大きく依存する
- Revit運用ルールとの接続が必要
- 3D表示までやると重くなるため、初期MVPは一覧表示でよい
- ライセンス確認必須
