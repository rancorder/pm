# Tech Card: Construction RAG

## 技術名

建設ドキュメントRAG / 図面・仕様書・議事録検索AI

## 参照候補

- LlamaIndex
- LangChain / LangGraph
- Haystack
- PDF table extraction tools
- Vector DB

## 何ができるか

図面、仕様書、議事録、見積、工程表、契約資料などを横断検索し、根拠付きで回答する。

## 解決できる顧客課題

- 資料探しに時間がかかる
- 図面変更と議事録の対応が追えない
- 過去案件の知見が再利用されない
- 仕様確認が属人化している
- PMが状況把握に時間を取られる

## Arent領域との親和性

- BIM: 高
- Revit: 中
- IFC: 中
- 配筋: 高
- プラント: 高
- 図面/仕様書RAG: 最高

## 入力

- PDF図面
- 仕様書
- 議事録
- Excel
- 画像
- BIM/IFCから抽出したテキスト

## 出力

- 質問への回答
- 根拠文書
- 該当ページ
- 確認すべき不足情報
- 関連タスク

## MVPでの使い方

限定された資料フォルダを読み込み、「この案件で未決の設計条件は？」「仕様変更があった箇所は？」などに根拠付きで答える。

## プロトタイプ案

```text
Document Upload
↓
Indexing
↓
Question Answering
↓
Source Evidence
↓
Action Items
```

## 実装難易度

中

## 注意点

- PDF品質に依存する
- 図面そのものの読解は段階的に進める
- 初期MVPはテキスト資料＋議事録から始めるのが安全
- 根拠表示がない回答は業務利用に向かない

## 顧客に聞くべき質問

1. 探すのに一番時間がかかる資料は何か？
2. よく聞く質問は何か？
3. 回答の根拠はどの粒度で必要か？

## エンジニアに聞くべき質問

1. 初期対象ファイル形式は何に絞るべきか？
2. OCRが必要か？
