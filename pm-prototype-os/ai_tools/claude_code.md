# ClaudeCode.md

PM Prototype OSにおけるClaude Codeの役割。

## 役割

Claude Codeは、既存リポジトリ理解・コード調査・修正方針作成・実装補助の担当。

主な用途は以下。

- 既存リポジトリの構造理解
- 重要ファイルの特定
- 技術スタックの把握
- バグ原因調査
- リファクタ方針作成
- 実装箇所の特定
- PR前の自己レビュー
- エンジニアへのコード観点共有

## 得意な仕事

```text
既存repo
↓
構造理解
↓
影響範囲特定
↓
修正方針
↓
安全な実装支援
```

## 使うタイミング

- 顧客や社内の既存システムrepoを読むとき
- プロトタイプを既存コードに組み込みたいとき
- どのファイルを触ればよいか分からないとき
- エラーが出て原因を追いたいとき
- OSSを取り込めるか調査したいとき
- 技術負債や設計リスクを確認したいとき

## 標準プロンプト：repo理解

```text
このリポジトリをPM視点で理解したいです。

# 確認してほしいこと
1. このプロダクトは何をするものか
2. 技術スタック
3. 主要ディレクトリ
4. 主要ファイル
5. データの流れ
6. API / DB / UI の関係
7. 変更すると危なそうな箇所
8. プロトタイプ追加に向いている箇所
9. PMが理解すべき技術リスク
10. エンジニアに確認すべき質問

# 注意
- 推測と事実を分ける
- ファイルパスを明示する
- 変更前に影響範囲を説明する
```

## 標準プロンプト：実装方針

```text
以下のPM要件を、このリポジトリに追加する場合の実装方針を作ってください。

# PM要件

# 出力
1. 追加・変更するファイル
2. 変更理由
3. 影響範囲
4. 実装ステップ
5. テスト観点
6. リスク
7. 先にPMへ確認すべきこと

# 制約
- いきなり大規模改修しない
- 既存設計を尊重する
- ダミー実装でよい箇所は分ける
- 本番化時に必要な追加対応を書く
```

## Claude Codeにやらせないこと

- 顧客秘密情報を含むrepoの外部公開
- ライセンス未確認OSSの無断コピー
- 破壊的変更の無確認実行
- `.env` や秘密鍵の表示・コミット
- 危険コマンドの実行

## PMとしての確認ポイント

- どのファイルを触るか明示されているか
- 影響範囲が説明されているか
- MVPに不要な作り込みをしていないか
- 顧客価値に関係ない技術遊びになっていないか
- エンジニアに渡せる粒度になっているか

## Claude Code向け安全ルール

```text
実装前に必ず以下を出す。

1. 現状理解
2. 変更方針
3. 変更対象ファイル
4. 影響範囲
5. リスク
6. 実装ステップ

その後、PM確認後に実装する。
```
