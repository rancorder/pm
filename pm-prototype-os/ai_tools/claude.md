# Claude.md

PM Prototype OSにおけるClaudeの役割。

## 役割

Claudeは、長文読解・議事録整理・仕様書化・文章生成の担当。

主な用途は以下。

- 長い議事録の整理
- 顧客資料の読み込み
- 仕様書の下書き
- 顧客向け説明文
- エンジニア向け依頼文
- 比較表・差分表の作成
- 文脈を保った要約

## 得意な仕事

```text
大量の文章・資料
↓
文脈理解
↓
抜け漏れ整理
↓
読みやすい文書化
```

## 使うタイミング

- 議事録が長いとき
- 顧客資料が多いとき
- Google Drive資料をまとめたいとき
- 要件定義書を読みやすくしたいとき
- エンジニアへの依頼文を丁寧にしたいとき
- 顧客に見せる資料の文章品質を上げたいとき

## 標準プロンプト：議事録整理

```text
あなたは建設・BIM・製造DXに強いPM補佐です。
以下の議事録を、要件定義に使える形で整理してください。

# 出力
1. 会議の目的
2. 顧客の発言要約
3. 顧客の課題
4. 表面的な要望
5. 背景事情
6. 未決事項
7. 決定事項
8. 次回確認事項
9. 要件定義に使える情報
10. プロトタイプ化できそうなテーマ

# 注意
- 顧客の発言と推測を分ける
- 不明点は不明と書く
- 技術的に断定しない
- PMが次に動ける形で整理する
```

## 標準プロンプト：仕様書化

```text
以下のPMメモを、エンジニアに共有できる仕様メモに整えてください。

# 出力
- 背景
- 解決したい課題
- 対象ユーザー
- ユースケース
- 機能要件
- 非機能要件
- MVP範囲
- 今回やらないこと
- 確認事項
- リスク
```

## Claudeにやらせないこと

- 実装の最終判断
- ライセンス判断の断定
- 不確かな技術情報の断定
- 顧客の意図を勝手に決めつけること

## PMとしての使い方

Claudeは文章化に強いが、PM判断は別途GPTや人間が行う。

Claudeの出力を使う前に確認する。

- 顧客発言と推測が混ざっていないか
- 文章がきれいすぎて論点がぼやけていないか
- 不明点が不明として残っているか
- 次アクションが明確か
