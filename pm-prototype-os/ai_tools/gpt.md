# GPT.md

PM Prototype OSにおけるGPTの役割。

## 役割

GPTは、PMの頭脳補助として使う。

主な用途は以下。

- 顧客ニーズの構造化
- 表面的な要望と根本課題の分離
- 要件定義
- MVP切り出し
- PM判断
- 顧客説明シナリオ
- 次回質問設計
- エンジニアへの論点整理

## 得意な仕事

```text
混沌とした情報
↓
構造化
↓
意思決定可能な論点
↓
次アクション
```

## 使うタイミング

- 顧客打ち合わせ直後
- 議事録から要件を作るとき
- 顧客の本当の課題を見抜きたいとき
- MVPを3案出したいとき
- エンジニアに渡す前に論点整理したいとき
- 会社提案の骨子を作るとき

## 標準プロンプト

```text
あなたは建設・BIM・プラント・製造DXに強いプロダクトマネージャーです。
以下の顧客情報から、PMとして次に判断すべき内容を整理してください。

# 入力
- 顧客議事録
- 顧客の発言
- 現在の業務
- 困っていること
- 既存ツール
- 制約条件

# 出力
1. 顧客の表面的な要望
2. 根本課題
3. 課題の構造
4. 現行業務フロー
5. 理想業務フロー
6. 要件一覧
7. MVP候補3案
8. 推奨MVP
9. 技術候補
10. 技術リスク
11. エンジニアに聞くべきこと
12. 次回顧客に聞くべきこと
```

## GPTにやらせないこと

- 実コードの大規模修正
- 顧客秘密情報を含む公開用文書作成
- ライセンス判断の断定
- 技術的に検証していない内容の断定

## PMとしての使い方

GPTの出力は答えではなく、意思決定の下書き。

必ず以下を確認する。

- 顧客の発言とズレていないか
- 技術的に盛りすぎていないか
- MVPが大きすぎないか
- エンジニアに渡せる粒度か
- 顧客に次回確認すべき不明点が残っているか
