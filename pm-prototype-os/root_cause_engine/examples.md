# Root Cause Engine Examples

## Example 01: 図面を検索したい

### 顧客発言

```text
図面をAIで検索したいです。
```

### 表面的な要望

図面検索AI。

### 真因候補

| 仮説 | 内容 | MVP |
|---|---|---|
| A | 図面の最新版が分からない | 図面版管理ダッシュボード |
| B | 図面内の情報が探せない | 図面RAG Evidence Viewer |
| C | 議事録の変更が図面へ反映されたか追えない | 変更ログ管理 |
| D | BIM属性が不足して検索できない | IFC属性欠損チェック |
| E | 若手が図面の見方を知らない | 図面Q&A / ルール検索 |

### 最初の質問

```text
探したいのは、図面ファイルそのものですか？
それとも図面内に書かれた情報ですか？
```

### 推奨MVP

最初は図面ファイル台帳＋版管理＋キーワード検索。

理由：

- データ構造化が先
- RAGより小さく検証できる
- 価値が見えやすい

### 実装案

```text
Next.js
shadcn/ui
CSV/JSON図面台帳
メタデータ検索
ステータス表示
```

## Example 02: BIMを活用したい

### 顧客発言

```text
BIMをもっと活用したいです。
```

### 表面的な要望

BIM活用ツール。

### 真因候補

| 仮説 | 内容 | MVP |
|---|---|---|
| A | BIM活用目的が決まっていない | BIMユースケース整理 |
| B | 属性情報が不足している | IFC属性欠損チェック |
| C | Revit操作が属人化している | 操作計画AIモック |
| D | モデルと図面/仕様書がズレる | 整合チェック候補一覧 |
| E | 維持管理へ引き継げない | 設備台帳RAG |

### 最初の質問

```text
BIMを活用したい目的は、設計効率化、積算、施工確認、維持管理のどれに一番近いですか？
```

### 推奨MVP

IFC属性欠損チェック。

理由：

- BIM活用の前提品質を見える化できる
- Arent領域と親和性が高い
- PoCとして小さく作れる

### 実装案

```text
IfcOpenShell
IFC to JSON
属性欠損ルール
Next.jsレビュー画面
CSVレポート
```

## Example 03: 見積比較が大変

### 顧客発言

```text
協力会社の見積比較が大変です。
```

### 表面的な要望

見積比較AI。

### 真因候補

| 仮説 | 内容 | MVP |
|---|---|---|
| A | 見積フォーマットが違う | 見積項目抽出 |
| B | 条件差が見えない | 条件差分比較表 |
| C | 数量根拠が不明 | BOQ差分チェック |
| D | 除外事項を見落とす | 除外条件抽出 |
| E | 過去単価と比較できない | 過去単価検索 |

### 最初の質問

```text
比較で大変なのは、金額差を見ることですか？
それとも条件差や除外事項を確認することですか？
```

### 推奨MVP

複数見積の条件差分テーブル。

### 実装案

```text
PDF/Excel extraction
structured table
difference review UI
clarification question generator
```

## Example 04: Revit操作を効率化したい

### 顧客発言

```text
Revit操作をもっと効率化したいです。
```

### 表面的な要望

Revit操作AI。

### 真因候補

| 仮説 | 内容 | MVP |
|---|---|---|
| A | 繰り返し操作が多い | コマンド履歴/再実行候補 |
| B | 操作手順が属人化 | 操作手順生成 |
| C | ファミリ管理が複雑 | ファミリ検索/分類 |
| D | ルール確認が大変 | 設計ルールRAG |
| E | 操作ミスが怖い | 操作計画＋確認画面 |

### 最初の質問

```text
効率化したいのは、繰り返し操作ですか？
それとも熟練者しか分からない判断や手順ですか？
```

### 推奨MVP

自然言語入力→操作計画→確認→実行ログモック。

### 実装案

```text
Command Console UI
intent classification
operation plan generator
approval step
mock execution log
```
