# Hypothesis Scoring

真因候補を感覚ではなく、PM判断に使える形で優先順位づけする。

## スコアリングの目的

顧客の要望に対して、複数の真因候補を出し、どれから検証するか決める。

```text
真因候補を広げる
↓
証拠で重みづけする
↓
切り分け質問を作る
↓
MVPで検証する
```

## スコア項目

| 項目 | 意味 | 1点 | 5点 |
|---|---|---|---|
| Evidence | 顧客発言・資料の根拠 | 根拠薄い | 明確な根拠あり |
| Impact | 解けた時の業務効果 | 小さい | 大きい |
| Frequency | 発生頻度 | 稀 | 毎日/毎案件 |
| Urgency | すぐ困っているか | 後回し可 | 今すぐ必要 |
| Feasibility | 検証しやすさ | 難しい | すぐ検証可 |
| Data Readiness | データがあるか | ない | すぐ使える |
| MVP Fit | 小さく見せられるか | 見せにくい | 1画面で見せられる |
| Arent Fit | Arent領域と合うか | 遠い | 直結 |

## 優先度計算

```text
Priority Score = Evidence + Impact + Frequency + Urgency + Feasibility + Data Readiness + MVP Fit + Arent Fit
```

最大40点。

## 判定

| 点数 | 判定 | 対応 |
|---:|---|---|
| 32-40 | S | 最優先でMVP化 |
| 25-31 | A | 条件確認後にMVP化 |
| 18-24 | B | 保留。追加質問が必要 |
| 17以下 | C | 今回は扱わない |

## 切り分け質問の作り方

高スコア仮説同士を分ける質問を優先する。

例：

```text
探したいのは「ファイルとしての最新版」ですか？
それとも「図面内に書かれた条件や部材」ですか？
```

この1問で、版管理系とRAG系を切り分けられる。
