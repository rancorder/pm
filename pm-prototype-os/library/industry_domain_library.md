# Industry Domain Library

建設業を一括りにせず、PM要件定義で使える粒度に分解するための業界別ドメインライブラリ。

## 使い方

顧客ヒアリング時に、まず対象領域を特定する。

```text
建築 / 土木 / プラント / 設備 / 電気 / 配管 / 鉄骨 / 配筋 / 維持管理
```

その後、以下を確認する。

- 主な業務フロー
- 主要データ
- よくある課題
- AI化ポイント
- MVP候補

---

## 01 建築

### 主な業務

- 企画
- 基本設計
- 実施設計
- 施工図作成
- 積算
- 施工管理
- 検査
- 引渡

### 主要データ

- 意匠図
- 構造図
- 設備図
- BIMモデル
- 仕様書
- 見積書
- 工程表
- 検査記録

### よくある課題

- 図面間の整合不備
- 設計変更の伝達漏れ
- BIM属性不足
- 施工図レビューの属人化
- 検査記録と写真の紐付け不足

### AI化ポイント

- 図面・仕様書RAG
- BIM属性チェック
- 議事録から設計変更抽出
- 施工図チェック支援

### MVP候補

- 図面・仕様書の不整合候補一覧
- BIM属性欠損チェック画面
- 設計変更ログ生成
