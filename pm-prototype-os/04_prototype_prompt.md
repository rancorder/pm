# 04 Prototype Prompt

Claude / Codex / Cursor / v0 / Lovable 等に渡すプロトタイプ生成プロンプト。

## 顧客議事録からPM成果物を作る

```text
あなたは建設・BIM・プラント・製造業のDXに強いプロダクトマネージャーです。
以下の顧客議事録から、要件定義とプロトタイプ方針を作成してください。

# 入力
- 顧客議事録
- 顧客資料
- 画面キャプチャ
- 図面/仕様書の概要
- 現在使っているツール

# 出力
1. 顧客が本当に困っていること
2. 表面的な要望と根本課題の分離
3. 現行業務フロー
4. 理想業務フロー
5. 要件一覧
6. MVP候補3案
7. 推奨MVP
8. 必要画面
9. 必要データ
10. 技術候補
11. 技術リスク
12. エンジニアに確認すべき質問
13. 次回顧客に確認すべき質問
14. プロトタイプ生成用プロンプト
```

## プロトタイプ生成プロンプト

```text
あなたは優秀なフロントエンドエンジニア兼プロトタイピング専門家です。
以下の要件から、顧客確認用のWebプロトタイプを作成してください。

# 目的
顧客が業務改善イメージを理解でき、次回打ち合わせで要件の差分を議論できる状態にする。

# 制約
- 実運用ではなくデモ用
- ダミーデータでよい
- ログイン不要
- UIは業務システムらしく簡潔に
- 画面遷移よりも業務フローが伝わることを優先
- 重要な判断ポイントが画面上で見えること

# 出力
- 画面構成
- コンポーネント構成
- ダミーデータ
- 実装コード
- 顧客に説明するポイント
```

## 建設/BIM向けUI観点

- 入力ファイル一覧
- 抽出結果
- 不整合候補
- 優先度
- 根拠表示
- 修正ステータス
- 担当者
- レポート出力

## PMレビュー観点

- 顧客が価値を判断できるか
- データの入口と出口が見えるか
- 業務フローが変わるポイントが見えるか
- エンジニアが実装難易度を見積もれるか
- 次回確認質問が自然に出るか
